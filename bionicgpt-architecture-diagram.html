üîê Authentication Flow

1
User Access
User navigates to https://chat.codemonkey.ai
2
Caddy Receives Request
Hecate's Caddy proxy receives the HTTPS request and queries Consul for bionicgpt service location
3
Forward to oauth2-proxy
Caddy forwards request over VPN tunnel to oauth2-proxy on local node
4
Check Authentication
oauth2-proxy checks for valid authentication cookie. None found ‚Üí redirect to Authentik
5
Authentik Login
User is redirected to https://auth.codemonkey.ai and logs in with credentials (+ MFA for superadmin)
6
Authorization Code
Authentik issues authorization code and redirects back to /oauth2/callback
7
Token Exchange
oauth2-proxy exchanges code for access token (server-to-server with Authentik over VPN)
8
Set Cookie & Forward
oauth2-proxy sets encrypted cookie and forwards to BionicGPT with headers:
‚Ä¢ X-Auth-Request-Email: henry@codemonkey.ai
‚Ä¢ X-Auth-Request-Groups: bionicgpt-superadmin
9
BionicGPT Response
BionicGPT creates/identifies user, assigns to teams based on groups, and returns response
10
User Access Granted
Response flows back through oauth2-proxy ‚Üí Caddy ‚Üí User. Subsequent requests use cookie.
